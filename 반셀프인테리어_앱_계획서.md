# 반셀프 인테리어 앱 서비스 기획서

## 1. 서비스 개요

**서비스명:** 내일의집

**핵심 가치:** 인테리어를 하는 사람들이 겪는 현실적인 문제를 해결하고, 공정마다 최적의 전문가와 자재를 연결해주는 올인원 인테리어 관리 앱

**타겟 유저:**
- 인테리어를 처음 시도하며 전문가 도움이 필요한 초보자
- 공정별로 전문가 참여 수준을 조절하고 싶은 반셀프 인테리어 유저
- 턴키 계약을 고려 중이나 적정 비용이 궁금한 유저

---

## 2. 핵심 기능 (MVP)

### 2.1 공정 계획 세우기
- 인테리어 공정 템플릿 제공 (철거 → 설비 → 방수 → 타일 → 목공 → 도배 → 마루 → 조명 → 가구)
- 유저가 직접 공정 순서를 커스터마이징
- 각 공정별 예상 소요 기간 자동 산출
- 공정 간 의존관계(선행 공정) 표시

### 2.2 비용 계산기
- 공정 추가/제거 시 실시간 예상 비용 업데이트
- 평수, 지역, 자재 등급별 비용 범위 제공
- 전문가 참여도별 비용 비교 (보조 시공 / 부분 시공 / 전담 시공)
- 총 예산 대비 현재 누적 비용 시각화 (프로그레스 바)

### 2.3 공정별 체크리스트 & To-Do
- 각 공정 단계별 필수 확인사항 자동 생성
- 오늘 현재 내가 처한 단계에서 신경써야 할 항목 하이라이트
- 체크 완료/미완료 추적
- 공정별 주의사항 및 꿀팁 제공

### 2.4 캘린더
- 공정 일정을 캘린더에 자동 배치
- 전문가 예약 일정 연동
- 자재 배송 예정일 관리
- 일정 지연 시 후속 공정 자동 조정

### 2.5 AI 질문 답변
- 인테리어 관련 질문에 AI가 즉시 답변
- 현재 진행 중인 공정 컨텍스트 반영
- 사진 업로드 → AI가 상태 진단 및 조언
- 자주 묻는 질문 데이터베이스 축적

### 2.6 간이 도면 만들기
- 방 크기 입력 → 2D 평면도 자동 생성
- 가구/설비 배치 드래그 앤 드롭
- 면적 자동 계산 (타일, 도배지, 마루 등 자재량 산출 연동)
- 사진 촬영 → AI 기반 도면 추출 (향후)

### 2.7 레퍼런스 찾기
- 스타일별/공간별/예산별 인테리어 레퍼런스 검색
- 오늘의집, 핀터레스트 등 외부 레퍼런스 연동
- 마음에 드는 레퍼런스 스크랩 & 무드보드 만들기
- AI 기반 유사 이미지 추천

### 2.8 공정별 기록하기
- 각 공정 진행 시 사진/영상/메모 기록
- 시공 전/후 비교 사진 자동 정리
- 사용한 자재/업체 정보 기록
- 실제 소요 비용 기록 → 예상 비용과 비교

### 2.9 블로그 글 자동 생성
- 기록한 사진과 메모를 기반으로 AI가 블로그 글 자동 작성
- 네이버 블로그/인스타그램 등 플랫폼별 포맷 지원
- 톤앤매너 선택 (친근한/전문적인/유머러스한)
- 원클릭 SNS 공유

## 3. 서브 기능

### 3.1 턴키 계약서 평가
- 계약서 사진 업로드 또는 텍스트 입력
- AI가 계약 조건 분석 (공정 범위, 자재 명세, 하자 보수 조건 등)
- 시장 평균 대비 가격 적정성 판단
- 빠진 항목/불리한 조건 경고
- 체크해야 할 질문 리스트 생성

---

## 4. 비즈니스 모델

### 4.1 Phase 1 - 유저 확보 (무료 유틸리티)
- 계획 세우기, 비용 계산기, 체크리스트 → **무료 제공**
- AI 질문 답변 → 기본 횟수 무료, 이후 구독
- 블로그 자동 생성 → 월 N회 무료, 이후 구독

### 4.2 Phase 2 - 전문가 매칭 (수수료 모델)
- 각 공정별 전문가(기사님) 프로필 및 포트폴리오
- 스케줄 확인 → 가능한 전문가 자동 매칭
- 견적 비교 기능
- 시공 완료 후 리뷰 시스템
- **수익:** 매칭 성사 시 중개 수수료 (전문가측 또는 양측)

### 4.3 Phase 3 - 자재 커머스 (마진 모델)
- 각 공정에 필요한 자재 자동 추천
- 도면 기반 필요 수량 자동 계산
- 자재 비교/구매 원스톱
- **수익:** 자재 판매 마진 또는 제휴 수수료

### 4.4 프리미엄 구독 (월 구독)
- AI 질문 무제한
- 블로그 자동 생성 무제한
- 상세 비용 분석 리포트
- 전문가 우선 매칭

## 5. 기술 스택 (제안)

| 영역 | 기술 |
|------|------|
| **프론트엔드** | React Native (iOS/Android 동시 지원) 또는 Flutter |
| **백엔드** | Node.js (NestJS) 또는 Python (FastAPI) |
| **데이터베이스** | PostgreSQL + Redis (캐싱) |
| **AI/LLM** | Claude API (질문답변, 블로그 생성, 계약서 분석) |
| **이미지 처리** | Claude Vision (사진 진단, 도면 추출) |
| **도면 에디터** | Canvas API / Fabric.js |
| **인증** | Firebase Auth 또는 Supabase Auth |
| **파일 저장** | AWS S3 또는 Cloudflare R2 |
| **푸시 알림** | Firebase Cloud Messaging |
| **결제** | 토스페이먼츠 / 아임포트 |

## 6. 개발 로드맵

### Phase 1: MVP (3~4개월)
| 주차 | 마일스톤 |
|------|---------|
| 1~2주 | 와이어프레임 & 디자인 시스템 확정 |
| 3~4주 | 회원가입/로그인, 프로젝트 생성 |
| 5~6주 | 공정 계획 세우기, 비용 계산기 |
| 7~8주 | 체크리스트, 캘린더 |
| 9~10주 | AI 질문 답변 연동 |
| 11~12주 | 공정 기록하기, 블로그 자동 생성 |
| 13~14주 | 턴키 계약서 평가 |
| 15~16주 | QA, 베타 테스트, 출시 |

### Phase 2: 성장 (출시 후 3~6개월)
- 전문가 매칭 시스템
- 도면 에디터
- 레퍼런스 검색 고도화
- 유저 커뮤니티 기능

### Phase 3: 수익화 (출시 후 6~12개월)
- 자재 커머스 연동
- 프리미엄 구독 모델 런칭
- B2B 인테리어 업체 대시보드

## 7. 핵심 화면 구성

```
[홈]
├── 내 프로젝트 (진행중인 인테리어)
│   ├── 공정 타임라인 (현재 단계 하이라이트)
│   ├── 오늘의 체크리스트
│   └── 예산 현황 요약
│
[공정] ← 기존 '계획'+'기록' 통합
├── 공정 리스트 (추가/삭제/순서변경)
│   └── 각 공정 상세
│       ├── 체크리스트 & To-Do
│       ├── 기록 (사진/영상/메모)
│       ├── 전후 비교
│       ├── 전문가 찾기 (견적 요청/매칭/예약)
│       │   ├── 전문가 프로필 & 포트폴리오
│       │   ├── 견적 비교
│       │   └── 시공 리뷰 & 평점
│       ├── 필요 자재 목록
│       └── 비용 (예상 vs 실제)
├── 비용 계산기 (전체 요약)
├── 캘린더
├── 도면
└── 블로그 글 생성 (기록 기반)

[쇼핑] ← 자재/커머스 특화
├── 자재 카테고리 브라우징
├── 인기 자재 & 추천
├── 자재 가격 비교
├── 공정별 필요 자재 추천
├── 도면 연동 수량 자동 계산
├── 장바구니 & 구매
└── 주문/배송 관리

[AI 도우미]
├── 채팅 (질문 답변)
├── 사진 진단
├── 계약서 분석
└── 레퍼런스 추천

[마이페이지]
├── 프로필
├── 완료된 프로젝트
├── 구독 관리
└── 설정
```

## 8. 경쟁 우위 & 차별점

| 기존 서비스 | 한계 | 우리의 차별점 |
|------------|------|-------------|
| 오늘의집 | 레퍼런스 중심, 실제 시공 관리 X | 계획→실행→기록 전 과정 관리 |
| 인테리어 견적 앱 | 견적만 제공, 시공 과정 지원 X | 공정별 전문가 매칭으로 최적 비용 실현 |
| 셀프 인테리어 카페 | 정보 파편화, 검색 어려움 | AI가 맥락 파악하여 즉시 답변 |
| 엑셀/노션 관리 | 수동 관리, 인테리어 특화 X | 인테리어 특화 자동화 도구 |

## 9. KPI 지표

**유저 획득:**
- MAU (월간 활성 유저)
- 프로젝트 생성 수

**유저 활성도:**
- 공정 기록 빈도
- AI 질문 횟수
- 체크리스트 완료율

**수익:**
- 전문가 매칭 전환율
- 구독 전환율
- 자재 구매 전환율 (Phase 3)

## 10. 리스크 & 대응

| 리스크 | 대응 방안 |
|--------|----------|
| 비용 데이터 정확도 | 실제 유저 데이터 축적 → 지역/시기별 보정 |
| 전문가 수급 | 초기 특정 지역 집중 → 점진적 확장 |

| 유저 이탈 | 인테리어 완료 후에도 유지보수 알림 등으로 리텐션 |
