# 반셀프 인테리어 사업계획서

---

## Executive Summary

**회사명:** (가칭) 내일의집
**서비스명:** 내일의집
**한 줄 요약:** 셀프 인테리어의 모든 과정을 AI로 관리하고, 필요한 순간에 전문가와 자재를 연결하는 올인원 인테리어 플랫폼

인테리어를 직접 해보려는 사람은 늘어나지만, 정보는 카페/블로그에 파편화되어 있고, 비용은 불투명하며, 공정 관리는 엑셀과 노션에 의존한다. 내일의집은 **계획 → 실행 → 기록**의 전 과정을 하나의 앱에서 해결하며, 축적된 유저 데이터를 기반으로 전문가 매칭과 자재 커머스로 수익을 창출한다.

---

## 1. 문제 정의 (Problem)

### 1.1 시장에서 발견한 페인포인트

| 문제 | 현재 유저의 대응 | 한계 |
|------|----------------|------|
| **공정 순서를 모름** | 네이버 카페에서 검색 | 글마다 답이 다르고, 내 상황에 맞는 정보 찾기 어려움 |
| **비용이 불투명** | 여러 업체 견적 비교 | 시간 소모 크고, 적정가인지 판단 불가 |
| **공정 관리가 안됨** | 엑셀/노션/수기 메모 | 인테리어 특화 기능 없음, 체크리스트 직접 작성 |
| **전문가 찾기 어려움** | 지인 소개/카페 추천 | 검증 안 됨, 스케줄 조율 번거로움 |
| **턴키 바가지 우려** | 경험자에게 물어봄 | 계약서 검토 전문지식 필요 |
| **기록이 남지 않음** | 사진만 찍어둠 | 정리 안 되고, 다음 시공 시 참고 불가 |

### 1.2 핵심 인사이트

> 셀프 인테리어 카페(네이버 "셀프인테리어" 카페 회원 100만+)에서 반복적으로 올라오는 질문의 80%는 **"이거 순서 맞나요?", "이 견적 적정한가요?", "이 단계에서 뭘 확인해야 하나요?"** 세 가지로 수렴한다.

→ 이 세 가지를 즉시 해결하는 도구가 곧 시장 진입의 핵심이다.

## 2. 시장 분석 (Market)

### 2.1 시장 규모

| 구분 | 규모 | 출처/근거 |
|------|------|----------|
| **국내 인테리어 시장 전체** | 약 60조원 (2025) | 한국건설산업연구원 |
| **주거 리모델링 시장** | 약 30조원 | 주거용이 전체의 약 50% |
| **셀프/반셀프 비중** | 약 4.5~6조원 (15~20%) | 1인 가구 증가, DIY 트렌드 |
| **TAM (전체 도달 가능 시장)** | 6조원 | 셀프/반셀프 인테리어 전체 |
| **SAM (유효 시장)** | 6,000억원 | 디지털 도구 활용 의향 있는 유저 (약 10%) |
| **SOM (초기 목표 시장)** | 120억원 | 수도권, 20~40대, 앱 적극 활용 유저 (SAM의 2%) |

### 2.2 시장 트렌드

- **1인 가구 증가:** 2025년 기준 전체 가구의 34.5% → 소규모 셀프 인테리어 수요 폭증
- **영끌 이후 인테리어 비용 절감 니즈:** 주택 구매 후 인테리어 예산 부족 → 반셀프 수요
- **콘텐츠화 트렌드:** 인테리어 과정을 블로그/유튜브로 기록하려는 니즈 증가
- **AI 도구 수용도 상승:** ChatGPT 이후 AI 활용에 대한 거부감 급감

### 2.3 경쟁 환경

| 서비스 | 포지션 | 강점 | 약점 | 우리와의 관계 |
|--------|--------|------|------|-------------|
| **오늘의집** | 인테리어 커머스 + 커뮤니티 | 트래픽, 브랜드, 커머스 | 시공 관리 도구 없음 | 간접 경쟁 (레퍼런스 영역) |
| **집닥** | 인테리어 업체 매칭 | 업체 네트워크 | 턴키 중심, 셀프 미지원 | 간접 경쟁 (매칭 영역) |
| **하우스앱** | 인테리어 견적 | 빠른 견적 | 공정 관리 없음 | 간접 경쟁 (견적 영역) |
| **네이버 카페** | 커뮤니티 | 방대한 정보량 | 정보 파편화, 검색 비효율 | 대체 관계 |
| **노션/엑셀** | 범용 도구 | 유연성 | 인테리어 특화 X | 대체 관계 |

**핵심:** 시공 "과정" 전체를 관리하는 전문 도구는 현재 시장에 부재하다.

## 3. 솔루션 (Solution)

### 3.1 제품 개요

**"인테리어의 모든 단계를 한 곳에서"**

```
┌─────────────────────────────────────────────────────┐
│                    반셀프 인테리어                      │
│                                                       │
│  [계획]          [공정 실행]                    [AI]    │
│  공정 설계       체크리스트 & 기록               질문    │
│  비용 계산       전후 비교 사진                  진단    │
│  도면           전문가 찾기 & 매칭              계약서   │
│  레퍼런스       자재 목록 & 구매                레퍼런스 │
│  캘린더         블로그 글 생성                          │
│                                                       │
│              ─── AI가 전 과정을 관통 ───                │
└─────────────────────────────────────────────────────┘
```

### 3.2 핵심 기능 상세

**1) 공정 계획 & 비용 계산기 (훅 기능 - 유저 유입의 핵심)**
- 평수/구조 입력 → 공정 템플릿 자동 생성
- 공정 추가/제거 시 **실시간 비용 변동** 시각화
- 셀프 vs 전문가 비용 즉시 비교
- "내 예산 2,000만원에 맞는 최적 조합" AI 추천

**2) 공정별 체크리스트 & AI 도우미**
- 현재 단계에서 꼭 확인해야 할 사항 자동 제공
- 사진 찍어서 올리면 AI가 상태 진단
- 24시간 인테리어 전문 AI 상담

**3) 턴키 계약서 AI 분석**
- 계약서 촬영 → 불리한 조항/누락 항목 즉시 경고
- 시장 평균 대비 가격 적정성 판단

**4) 공정별 기록 & 전문가 매칭**
- 공정별 사진/메모 타임라인 자동 정리
- 각 공정에서 바로 전문가 찾기 → 견적 비교 → 매칭 → 예약
- AI가 블로그 글 자동 작성 → 원클릭 공유

### 3.3 유저 시나리오

> **김민수 (32세, 첫 자가 구매):** 25평 아파트를 구매했지만 인테리어 예산이 빠듯하다. 도배/마루는 직접 하고, 설비/타일은 전문가에게 맡기고 싶다.
>
> 1. 앱에서 25평/아파트/반셀프 선택 → 공정 계획 자동 생성
> 2. 도배/마루를 "셀프"로, 설비/타일을 "전문가"로 설정 → 예상 비용 1,200만원 확인
> 3. 각 공정 화면에서 체크리스트 확인 + 사진/메모 기록을 동시에 진행
> 4. 설비 공정 상세 → "전문가 찾기" 탭에서 바로 견적 비교 → 매칭 → 예약
> 5. 막힐 때 AI에 질문, 사진 찍어서 진단 받기
> 6. 모든 공정 완료 → 축적된 기록으로 블로그 자동 생성 → 후기 공유

## 4. 비즈니스 모델 (Business Model)

### 4.1 수익 구조

```
                    수익 모델 3층 구조

    ┌──────────────────────────────────────┐
    │    Layer 3: 자재 커머스 (Phase 3)      │  GMV 수수료 5~15%
    ├──────────────────────────────────────┤
    │    Layer 2: 전문가 매칭 (Phase 2)      │  건당 중개 수수료 10~15%
    ├──────────────────────────────────────┤
    │    Layer 1: SaaS 구독 (Phase 1)       │  월 9,900~19,900원
    ├──────────────────────────────────────┤
    │    Foundation: 무료 유틸리티           │  유저 확보 & 데이터 축적
    └──────────────────────────────────────┘
```

### 4.2 수익 모델 상세

| 수익원 | 과금 방식 | 예상 단가 | 시작 시점 |
|--------|----------|----------|----------|
| **프리미엄 구독** | 월 정기결제 | 9,900~19,900원/월 | MVP 출시 시 |
| **전문가 매칭 수수료** | 매칭 성사 건당 | 시공비의 10~15% | 출시 후 6개월 |
| **자재 커머스** | 구매 수수료/마진 | GMV의 5~15% | 출시 후 12개월 |
| **광고/제휴** | 자재 브랜드 노출 | CPM/CPC | 출시 후 12개월 |

### 4.3 무료 vs 프리미엄

| 기능 | 무료 | 프리미엄 (월 9,900원) | 프로 (월 19,900원) |
|------|------|---------------------|-------------------|
| 공정 계획 | 1개 프로젝트 | 3개 프로젝트 | 무제한 |
| 비용 계산기 | 기본 | 상세 리포트 포함 | 상세 리포트 + AI 최적화 |
| 체크리스트 | O | O | O |
| AI 질문 | 일 3회 | 일 30회 | 무제한 |
| 블로그 생성 | 월 1회 | 월 10회 | 무제한 |
| 계약서 분석 | X | 월 3회 | 무제한 |
| 전문가 매칭 | 기본 | 우선 매칭 | 우선 매칭 + 전담 매니저 |

### 4.4 단위 경제 (Unit Economics) 추정

| 지표 | 수치 | 근거 |
|------|------|------|
| **CAC (고객 획득 비용)** | 15,000원 | 인테리어 카페/SNS 마케팅 기반 |
| **구독 전환율** | 5% | 유틸리티 앱 평균 3~7% |
| **ARPU (유저당 월 매출)** | 1,500원 | (무료 95% x 0) + (구독 5% x 30,000원 가중평균) |
| **평균 이용 기간** | 4개월 | 인테리어 프로젝트 평균 기간 |
| **LTV (고객 생애 가치)** | 6,000원 (구독만) | 구독 전환 기준 |
| **LTV (매칭 포함)** | 45,000원 | 매칭 전환율 10%, 건당 수수료 30만원 |
| **LTV/CAC** | 3.0x (매칭 포함) | 건전한 수준 |

## 5. Go-to-Market 전략

### 5.1 유저 획득 전략

**Phase 1: 커뮤니티 침투 (0~6개월)**
| 채널 | 전략 | 예상 효과 |
|------|------|----------|
| **네이버 셀프인테리어 카페** | 비용 계산기/체크리스트를 무료 도구로 바이럴 | 초기 유저 1만명 |
| **인스타그램/유튜브** | 셀프 인테리어 과정 콘텐츠 (앱 활용 사례) | 브랜드 인지도 |
| **네이버 블로그 SEO** | "셀프 인테리어 비용", "인테리어 순서" 키워드 점령 | 오가닉 유입 |

**Phase 2: 입소문 & 콘텐츠 루프 (6~12개월)**
- 유저가 만든 블로그 글 → 자연 검색 유입 → 앱 다운로드 (바이럴 루프)
- 완료된 프로젝트 공유 → 새 유저가 레퍼런스로 활용

**Phase 3: 전문가 네트워크 효과 (12개월~)**
- 전문가가 많을수록 유저 유입 ↑ → 유저가 많을수록 전문가 유입 ↑
- 양면 네트워크 효과 확보

### 5.2 초기 집중 전략

- **지역:** 수도권 (서울/경기) 집중
- **세그먼트:** 20~30평대 아파트 반셀프 인테리어
- **킬러 기능:** 비용 계산기 + 체크리스트 (무료 훅)

## 8. 기술 전략

### 8.1 기술 스택

| 영역 | 선택 | 선택 이유 |
|------|------|----------|
| **모바일** | React Native | 크로스플랫폼, 개발 속도, 인력 풀 |
| **백엔드** | NestJS (TypeScript) | 프론트와 언어 통일, 타입 안정성 |
| **DB** | PostgreSQL + Redis | 안정성, 확장성 |
| **AI** | Claude API | 긴 컨텍스트 처리 (계약서 분석), 한국어 성능 |
| **인프라** | AWS (초기 Supabase 활용) | 확장성, 한국 리전 |
| **도면** | Fabric.js + Canvas API | 웹 기반 2D 에디터 |

### 8.2 데이터 경쟁력

축적되는 데이터가 곧 해자(moat):
- **비용 데이터:** 지역/평수/시기별 실제 시공 비용 → 정확도 향상
- **공정 데이터:** 실제 소요 기간, 자주 발생하는 문제 → 체크리스트 고도화
- **전문가 데이터:** 시공 품질, 리뷰 → 매칭 정확도 향상
